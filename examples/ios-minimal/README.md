# ios-minimal

Phase 0 verification: minimal iOS app that uses the `LyraRuntime` Swift
Package and shows "Hello, Lyra" in the Simulator.

No Rust, no Lynx — only Swift + UIKit + the local `LyraRuntime` SPM package.
This proves the SPM distribution path works before we layer in the heavier
binary frameworks (Lynx xcframework, Rust xcframework, C++ bridge xcframework)
in later phases.

## Prerequisites

- macOS with Xcode 15 or later
- [XcodeGen](https://github.com/yonaskolb/XcodeGen):
  ```sh
  brew install xcodegen
  ```

## Build & run

```sh
# 1. Generate the Xcode project from project.yml
xcodegen generate

# 2. Open in Xcode and run on a simulator
open HelloLyra.xcodeproj
# Pick an iOS Simulator destination, hit ⌘R
```

Alternatively, command-line build only:

```sh
xcodegen generate
xcodebuild \
  -project HelloLyra.xcodeproj \
  -scheme HelloLyra \
  -destination 'generic/platform=iOS Simulator' \
  build
```

## What you should see

A blank white (or system-background-colored) screen with "Hello, Lyra"
centered in 32pt semibold text.

## Files

- `project.yml` — XcodeGen spec. Regenerate `HelloLyra.xcodeproj` after
  any change.
- `Sources/AppDelegate.swift` — extends `LyraAppDelegate` from the SPM
  package. That base class handles window setup and root view controller.
- `Info.plist` — generated by XcodeGen from `project.yml`.

## Known phase limitations

- No Rust — there is no Rust runtime loaded; `LyraView`'s native callbacks
  are stubbed.
- No Lynx — `LyraView` is currently a plain `UIView`, not a `LynxView`
  subclass. The placeholder UILabel is plain UIKit.
- No CNG — this app's structure is hand-written, not generated by `lyra prebuild`.
